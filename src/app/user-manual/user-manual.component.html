1. Add Database

1.1: Create a map called 'TestCSV'
1.2: Put Clinical1 in the map 'TestCSV'
(Clinical1.csv Clinical1.csv Location)
1.3: Go to Database
1.4: Press 'Add Database'
1.5: Put 'TestCSV' as the databasename
1.6: Put the url of the created map in url (../TestCSV)
1.7: Select 'CSV' as Format
1.8: Click on 'Save Database'
*If you see TestCSV in the table on the database page it worked

2. Add Mapping

2.1: Go to Mapping
2.2: Download mapping.ttl from mapping.ttl Location
2.3: Put mapping.ttl in the choose file area
2.4: Click on 'Upload Mapping'
2.5: Select a format on the left side of the screen
2.6: Press the button 'Download current mapping'
*If everything went correct you should be downloading the file you just uploaded

3. Upload Ontology

3.1: Go to Ontology
3.2: Click on 'Add Ontology'
3.3: Click on the checkbox next to url
3.4: Put the following in the textfield
http://data.bioontology.org/ontologies/ROO/submissions/8/download?apikey=8b5b7825-538d-40e0-9e9e-5ab9274a9aeb
3.5: Fill in http://www.cancerdata.org/roo/ in the base Uri box
3.6: Select ‘RDFXML’ as format
3.7: Press the ‘Save Ontology’ Button
*If you did it correct you will see the ontology file in the table underneath the first button

4. Validate Database

4.1: Go to Validate Database
4.2: Change the SQL to 'SELECT * FROM Clinical1'
4.3: Put 'TestCSV' in the database part
4.4: Put a max results
4.5: Click on test Sql Query
4.6: Click on save Sql Query
*If everything went correct you should be able to go and use Validate Mapping and use this page
